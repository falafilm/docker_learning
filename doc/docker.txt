DEPLOYMENT

- Traditional Deployment 
	ติดตั้งรันแอพพลิเคชั่นโดยตรงใน os ของ serverจริง เช่น ติดตั้ว Python & MySQL ลง server แล้วรันแอพโดยตรง (localhost)
    
     ข้อเสีย >> เวลารันแอพหลายแอพและใช้ lib version ต่างกัน เช่น python 3.8 อีกอัน 3.11 ทำให้รันไม่ได้ในเครื่องเดียว 
	    >> ไม่มี Isolation (แยกทรัพยากรและสภาพแวดล้อมของแอพออกจากกัน)

	ประกอบด้วย : Hardware, Host OS, VM

- Virtualized Deployment (เปลืองทรัพยากร)
	เกิด "Software Hypervisor"(แยกทรัพยากร) จำลอง เครื่องคอมพิวเตอร์ (Virtual Machine : VM) หลายๆเครื่อง ขึ้นมาจากเครื่องจริง (Physical Server: Host) เพื่อรันหลายแอพในเครื่องเดียวกัน สามารถสร้างแอพแยกออกจากกันได้อิสระ
	ประกอบด้วย : Hardware (cpu, ram, disk), Host OS, Hypervisor(software ตัวกลางทำหน้าที่จัดการ ควบคุม VM แบ่งทรัพยากรให้แต่ละเครื่อง), VM

	>> Hypervisor(software ตัวกลางทำหน้าที่จัดการ ควบคุม VM แบ่งทรัพยากรให้แต่ละเครื่อง)
		- Bare-matal Hypervisor (Type-1) Hypervisor ที่ติดตั้งและทำงานบนฮาร์ดแวร์โดยตรง เช่น  VMWare ESX, Microsoft Hyper-X สำหรับใช้งานขนาดใหญ่ระดับองค์กร 
			?? Data Center และ Enterprise Server เป็นต้น ??

		- Hosted Hypervisor (Type-2) ที่ติดตั้งและทำงานบนระบบ OS เช่า VMware Workstation, Oracle Virtual Box เหมาะกับใข้งานทั่วไป 
			?? พัฒนาทดสอบโปรแกรม, รันโปรแกรมเก่า, Sandbox Environment ??

	>> VM ประกอบด้วย
		1. Guest OS >> ระบบปฏิบัติการที่ติดตั้งใน Windows, Linux
		2. Binaries / Libraries >> ชุดคำสั่งที่ใช้รัน

	1. docker -v /docker --version/ docker version

	3. docker system df แสดงข้อมูลการใข้พื้นที่ disk

	>> docker image
		1. docker pull <image-name>  --> download image เวอร์ชั่นล่าสุดจาก docker hub (ex. socker pull python)
		2. docker pull <image-name>:<tag> --> download image ตาม version (tag) ที่ระบุใน docker hub (ex. docker pull python:3.12-slim)
		3. docker images แสดงรายการ images ทั้งหมด

		6. docker rmi <image-name>:<tag> ลบ image ตาม ชื่อและ version
		7. docker image prune การลบ image ที่ไม่ได้ใช้ หลายรายการ

>> คำสั่งจัดการ Container
	1. docker run <image-name> รัน container จาก image  (รันครั้งแรกจะ pull image ให้เอง) --> docker run postgres
	2. docker run -d <image-name> รัน container แบบ background ไม่มี Log แสดง
		-d : detached ใข้ในงาน production 
	3. docker run -it <image-name> รัน container แบบ interactive พร้อม shell เข้าใช้งานภายใน container (ป้อนคำสั่งใน terminal) --> exit
	4. docker run --name <container-name><image-name> รัน container พร้อมกำหนดชื่อ container ด้วยตัวเอง

		** docker run -it --name <container-name><image-name> คือการรันแบบ powershell และสร้างชื่อเองได้ **
		** docker run --rm -it --name <container-name><image-name> หากหยุดรันทำการลบทันที **
		** docker run -dit --name test3 python **

	5. docker ps แสดงรายการ container ที่กำลังทำงานอยู่
	6. docker ps -a แสดงรายการ container ทั้งหมด 
	7. docker container prune ลบ container ที่หยุดทำงานออก
	8. docekr start <container-id> สั่งเริ่มต้น container

		** docker start -ai 2fbf เริ่มทำงาน -ai(attach + interactive) เพื่อให้โปรแกรมข้างในไม่ดับทันที**

	9. docker stop <container-id> หยุดทำงาน container
	10. docker restart <container-id> 
	11. docker pause <container-id> หยุดการทำงานชั่วคราว
	12. docker unpause <container-id> เริ่มทำงานต่อ
	13. docker rm <container-id> ลบ container ที่หยุดทำงาน
	14. docker rm -f <container-id> ลบ container แบบบังคับ
	15. docker rename [ชื่อเดิม_หรือ_IDเดิม] [ชื่อใหม่ที่ต้องการ]

>> Docker file
	สร้าง image จาก doockerfile โดยใช้คำสั่ง docker builde image ขึ้นมา
	image จะสร้าง layer ขึ้นมา โดยชั้นแรก คือ Base layer (Base image เอา image อื่นมาใช้ในการสร้าง image ใหม่่) ตามด้วย ลำดับการทำงานที่ระบุ(Intermediate layer) 

	1> Base Layer --> คำสั่ง FROM ใช้สำหรับ OS จุดเริ่มต้น image 
	2> Intermediate Layer --> คำสั่ง RUN, COPY, ADD, WORKDIR ใช้สำหรับการเพิ่มไฟล์, ติดตั้ง Library, คัดลอกไฟล์
	3> Metadata Layer --> คำสั่ง CMD, EXPOSE, ENV, LABEL ไม่มีการเพิ่ม แก้ไขหรือลบไฟล์ ทำให้มีขนาด 0 ไบต์ (เป็นแค่การตั้งค่า image )

	คำสั่ง
		> FROM <image> กำหนด base image (**อยู่บรรทัดเเรกเสมอ**) --> FROM pythom:3.13
		> WORKDIR <path> กำหนด folderทำงาน ของ container --> WORKDIR docker_learning/app
		> COPY <host-path><container-path> (ต้นทาง, ปลายทาง)คัดลองไฟล์จากเครื่องเราไป container --> COPY app.py docker_learning/app/ (*คัดลอกapp.py ไปยัง docker_learning/app/*)
		> RUN <command> ติดตั้ง dependencies ลงใน container --> RUN npm install, RUN pip install -r requiment.txt
		> EXPOSE <port-number> เปิด port ภายใน container --> EXPOSE 8080, EXPOSE 5000
		> CMD ["<command>", "<argl>"] ชุดคำสั่งที่ถูกเรียกใช้งานในตอนเริ่ม เมื่อรัน container --> CMD ["python", "app.py"], CMD ["node", "app.js"]

	ขั้นตอนการทำงาน
		1. กำหนด base image ที่ต้องการใช้ --> FROM python:3.13
		2. สร้าง folder ใน container สำหรับเก็บไฟล์งาน --> WORKDIR /Sudarat/app  (folder ที่ตั้งใน container)
		3. คัดลอกไฟล์ erequirement.txt เข้าไปที่ container --> COPY requirement.txt . (. คือ folder ที่จะสร้างขึ้นเพื่อเอาไปเก็บ)
		4. ติดตั้ง package ด้วย pip --> RUN pip install --no-cache-dir -r requirement.txt  (เป็นการล้าง cache เพื่อลดขนาดไฟล์ Image )
		5. คัดลอกไฟล์โค้ดทั้งหมดจาก host เข้าไปใน container  --> COPY .(space).  ให้ไปเก็บจาก docker_learning(Host foder ที่เปิดในปัจจุบันในเครื่องเราเอง)ในfolder /Sudarat/app(direct container) 
		6. ระบุ port ที่จะใช้ --> EXPOSE 5000
		7. กำหนดคำสั่งเริ่มต้น เมื่อเริ่มรัน container --> CMD ["python", "app.py"]
		8. สร้าง image: docker build -t <image-name> . (มีจุด)

			**docker build -t my-python-app .**

		9. สร้าง Image พร้อม tag: docker build -t <image-name>:<tag>

			**docker build -t my-python-app:0.1 .**
